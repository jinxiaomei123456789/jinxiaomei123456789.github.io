前言

  技术方面除了前面的物理安全和网络，还有主机、应用以及数据。新建一篇文章写吧，接着头一篇文章写都有一本书那么多了。
  # 主机安全-linux #
  身份鉴别
  a)应对登录操作系统和数据库系统的用户进行身份标识和鉴别。
  检查内容：1、密码文件中的口令字段是否不为空；2、检查各个用户主目录下的.rhosts文件；3、查看/etc/hosts.equiv。
  检查方法：1、查看/etc/passwd文件中各用户的第二字段处是否不为空；2、输入：find / -name “.hists”，查看各个.rhosts文件；3、查看/etc/hosts.equive。
  推荐值：1、第二字段是否不为空；2、应删除.rhosts和/etc/hosts.equiv中的用户或主机。
  判断标准：1、2都符合结果才能符合。
  b)操作系统和数据库系统管理用户身份标识应具有不易被冒用的特点，口令应有复杂度要求并定期更换；
  检查内容：1、密码最长有效期PASS_MAX_DAYS;2、密码最短存留期PASS_MIN_DAYS;3、密码长度最小值PASS_MIN_LENS;4、密码有效期警告PASS_WARE_AGE;5、密码须包含大写字母个数ucredit；6、密码须包含小写字母个数lcredit;7、密码须包含的数字字符个数dcredit;8、密码须包含的特殊符号个数ocredit。
  检查方法：1、查看/etc/login.defs,访谈询问当前所设置的密码长度及更换周期；2、查看/etc/pam.d/system-auth，确认密码复杂度要求。
  推荐值：1、PASS_MAX_DAYS:90;PASS_MIN_DAYS:2;PASS_MIN_LENS:8;PASS_WARN_AGE:7;2、password requisite pam_cracklib.so minlen=8 ucredit=-1 lcredit=-1 dcredit=-1 ocredit=-1；3、当前所设置的密码长度应不少于8位，具有一定的复杂度并定期更换。
  c)应启用登录失败处理功能，可采取结束会话、限制非法登录次数和自动退出等措施。
  检查内容：查看是否使用pam_tally.so模块跟踪不成功的登录次数。
  检查方法：1、查看/etc/pam.d/system-auth文件。
  推荐值：1、是否有如下参数（X为锁定次数，该参数需手动添加）account required /lib/security/pam tally.so deny=X no_magic_root reset。
  d)当对服务器进行远程管理时，应采取必要措施，防止鉴别信息在网络传输过程中被窃听。
  检查方法：1、询问并验证服务器是否关闭了Telnet、FTP等服务，是否使用SSH加密协议进行远程登录和管理（telnet服务查看/etc/xinetd/krd5-telnet文件，较低版本为inetd;rpm -aq|grep ssh查看是否安装了SSH相应的包）；2、输入netstat -an或lsof -i:21、lsof -i:22、lsof -i:23等命令查看21（FTP）、22（SSH）、23（Telnet）端口的运行情况。
  推荐值：1、disable=yes未开启Telnet服务；2、显示所安装的SSH包，采用SSH进行远程管理。
  判断标准：1、2均需符合。
  e)应为操作系统和数据库系统的不同用户分配不同的用户名，确保用户名具有唯一性。
  检查内容：查看是否存在用户名相同的帐户，是否存在同UID的用户。
  检查方法：查看/etc/passwd文件中是否有用户名相同、UID（第三字段相同的帐号）。
  推荐值：1、无同名帐号；2、UID具有唯一性；3、UID=0只能为root用户。
  判断标准：1、2、3均需符合。
  f)应采用两种或两种以上组合的鉴别技术对管理用户进行身份鉴别。
  检查内容：操作系统是否采用口令+令牌、USB、KEY等方式进行身份鉴别。
   访问控制
  a)应启用访问控制功能，依据安全策略控制用户对资源的访问。
  检查方法：询问系统管理员是否建立用户权限对照表，所设置的用户权限是否与权限表中一致。
  b)应根据管理用户的角色分配权限，实现管理用户的权限分离，仅授予管理用户所需的最小权限。
  检查内容：1、应在操作系统内建立下述几类帐户：管理员、审计员、操作员；2、有各类帐户对应的用户权限分配表。
  检查方法：1、查看/etc/passwd文件中各帐户并询问其用途、类别；2、检查用户权限分配表。
  1、按最小授权原则分配，不授予一般用户管理员权限。建立了权限分离的用户，如管理员、审计员、操作员等；2、建有各类帐户的用户权限分配表。
  c)应实现操作系统和数据库系统特权用户的权限分离。
  检查内容：操作系统和数据库系统管理员是否为同一自然人。
  d)应严格限制默认帐户的访问权限，重命名默认帐户，修改这些帐户的默认口令。
检查方法：查看是否对系统默认帐户进行重命名，询问并结合操作系统扫描结果确认是否存在默认口令帐户。
  推荐值：1、重命名默认帐户；2、修改默认帐户的口令。
  e)应及时删除多余的、过期的账户，避免共享帐户的存在。
  检查内容：是否删除多余过期的帐户。
  检查方法：查看/etc/shadow文件，询问相应帐户是否为过期、多余帐户（已锁定帐户其所在行第一个和第二个冒号之间的口令字段为“！！或”）
  经查看/etc/passwd配置文件和访谈，系统中不存在多余、过期账户。
  f)应对重要信息资源设置敏感标记。
  推荐值：1、定义了主机中的重要信息资源；2、为主机内的重要信息设置额敏感标记。
  g)应依据安全策略严格控制用户对有敏感标记信息资源的操作。
  推荐值：1、定义了敏感标记资源的访问策略；2、依据访问策略控制用户对有敏感标记重要信息资源的操作。
  安全审计
  a)审计范围应覆盖到服务器和重要客户端上的每个操作系统用户和数据库用户。
  检查内容：查看操作系统审计进程是否开启，如采用了第三方审计系统则查看系统运行是否正常。
  检查方法：输入service syslog status;service auditd status。
  审计进程运行正常。
  b)审计内容包括重要用户行为、系统资源的异常使用和重要系统命令的使用等系统内重要的安全相关事件。
  检查方法：1、输入cat /etc/syslog.conf;2、输入auditctl -s查看audit运行状态；auditctl -l查看audit规则。
  推荐值：1、syslog.conf应包含类似于如下值：
  .info;mail.none;news.none;authpriv.none;cron.none 2、/var/log/messages;AUDIT_STATUS:enabled=1,如配置规则，应有类似如下返回值：LIST_RULES:exit,always watch=/etc/hosts perm=rwa key=host-file。
  c)审计记录应包括事件的日期、时间、类型、主体标识、客体标识和结果等。
   检查方法：Linux系统审计默认符合，如使用第三方审计系统则查看审计日志是否包含前述信息。
  d)应能够根据记录数据进行分析，并生成审计报表。
  检查方法：询问并检查是否对审计日志进行定期查看、分析，生成审计分析报表。
  推荐值：1、应人工分析；2、数据制定日志记录表，且能生成审计报表。
  判断标准：1、2均符合结果为符合。
  e)应保护审计进程，避免受到未预期的中断。
  检查内容：询问是否安装额外的审计进程保护软件，确保审计进程不会受到未预期的中断。
  f)应保护审计记录，避免受到未预期的删除、修改或覆盖等。
  检查方法：1、查看syslog.conf\audit.conf文件中日志信息所在文件的访问权限，如:ls -l /var/log/messages;ls -l /var/log/secure;ls -l /var/log/audit/audit.log；2、访谈并询问是否对审计日志进行保护。
  推荐值：1、日志文件读写权限的推荐值可参考如下所示： ls -l /var/log/messages:640;ls -l /var/log/secure:640;ls -l /var/log/audit/audit.log:640;文件属主必须为审计类型帐户，且删除、修改或覆盖等操作会进行记录；2、采取如部署日志服务器、第三方行为审计系统、堡垒机保存操作日志或其他相应手段。
